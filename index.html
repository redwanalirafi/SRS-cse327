1. Introduction


1.1 Purpose

This Software Requirements Specification document describes and explains all the requirements that will be implemented in Virtual Classroom (1.0). The requirements are gathered after agreeing with the clients in the meetings and the requirements are written in a way so that anybody from the clients or developers can understand without googling. The product will support multiple target platforms (Web and Android). Virtual Classroom is a cross platform product where teachers and students can interact with each other and continue their educational journey in any situation. Teacher can upload PDF flies, assign students to read that, monitor them from derived data, evaluate them through exams. On the other hand, students can see which courses they are enrolled in, read the PDF files, and appear in exams. This platform will enable them to have the flavor of physical classroom with the convenience of staying at home without bearing the hazards and expenses of commuting. To encourage in distance learning and keep the students and teachers connected through a secured platform which is easy to use is one of the primary goals of this platform.

1.2 Intended Audience and Reading Suggestions

The document is intended for:

Each section is written in a way so that different users will read different sections. Reading each section will not come handy for every audience. The first two section is tailored for all audience where technical terms are absent. The next four sections use technical language which is intended for mainly the developers and testers. Every requirement will be detailed one by one. Good to know that more important requirements will be described in high details and less important requirements will be described in brief.

1.3 Product Scope

Virtual-Classroom is a platform where teachers and students will be connected in a secured way. Teachers and students will be able to continue their excellence journey even in any national crisis or pandemic through this platform. Moreover, education should be accessible for all. In the rural areas where quality teachers are absent, this platform could play a great role to nourish the students of that rural areas. Teachers can provide reading materials and students can read it. For evaluation, there will be exams and in which section students are performing well and in which sections students are confused, a report of that data will be presented to the teachers so the teachers can take necessary steps. Moreover, educational institutions can use this platform to enhance their quality of education as well. The web and android app we are developing for cross-platform use is an easily maintainable for developers and capable of handling large scale of data for the users.

2. Overall Description


2.1 Product Perspective

Connecting the students and teachers in a secured platform is the main driving force of specifying this project. Educational institutions need a platform where they will be able to share the reading materials with their students to accelerate their excellency. Moreover, to investigate where the students are struggling or confused, this platform will come handy. To create a classroom like environment for the users there are features to accomplish that goal.

2.2 Product Functions

  • Sign-Up using verified email id and password
  • Sign-In to existing account
  • Creating cohorts
  • Uploading PDF flies and set time for reading session
  • Monitor attentiveness
  • Setting questions for exams
  • Retrieving exam marks along with the report where students are struggling with
  • Viewing the cohorts
  • Reading PDF flies
  • Viewing the reading task and the time limit for reading session
  • Take exams and answering questions using voice
  • Notification about reading tasks when they are assigned
  • Log-Out option.

2.3 User Classes and Characteristics

This product will include two categories of users:
(i)Teachers:

  • Through Sign-Up individual account will be created for the teachers. Any teacher who has an email-id will be able to create an account by putting password in the password section.
  • Through Sign-In, the teachers can go the dashboard.
  • Teacher can make cohorts which consist of several students.
  • Teacher can upload PDF files and set the time for the students to read it.
  • Teacher can find how attentively the students are reading through a data which will be derived by some functions with the aid of camera during the reading session.
  • Teacher can find how attentively the students are reading through a data which will be derived by some functions with the aid of camera during the reading session.
  • After exams, teacher can see the marks of the students and in which part of which PDF files they are struggling with.
  • Teacher can log out from the platform.

(ii)Students:

  • Through Sign-Up individual account will be created for the students. Any student who has an email-id will be able to create an account by putting password in the password section.
  • Through Sign-In, the students can go the dashboard.
  • Students can view in which cohorts they are assigned.
  • Students can see the list of reading tasks and the time limit for reading the PDF flies which are assigned by the teachers.
  • Students can take part in exams, and they will have to answer the questions using voice.
  • Students will get notification when new reading task is assigned.
  • Students can log out from the platform.

2.5 Design and Implementation Constraints

We have a web application and an android app, but no IOS application for the system. IOS users can use the browser to access the web version of this system. Users can only take the exam through the android app. Users cannot use other features inside the mobile app. Browsers from any platform have functional access.

2.6 User Documentation

The enrolled students and faculty will be provided with printable manuals through email. The system will also provide an online help feature where students or faculty can ask any question. The faculty or student can ask for help to better understand the system.

2.7 Assumptions and Dependencies

Users must have a stable and working internet connection. Web users must have a working browser to use the product. Users must have a good quality web camera, and the camera must be installed in a good position where it shows the face correctly. Mobile users must have android version 6+ installed. The database will be modified to meet the system's requirements. New entities and relations for the user will need to be added.



3. External Interface Requirements


3.1 User Interfaces

The users will start at the login page, where they will find a button for google login. Users can log in using a valid google account. After a faculty member logs in, they can create an exam cohort where they can add other users (students). Inside the cohort, the user (evaluator) can create assessments and add questions in the form of MCQs, short questions, and micro-viva. After setting the assessment, the students can sit for the exam. The system will use face-recognition technology to identify students. Using the web camera, the system will track the eyes of the student and detect cheating. After the due time of the exam, the system will automatically calculate the marks, and the evaluator will be able to download the mark sheet of each student. The system will analyze the script and identify the student's weakness and the evaluator/faculty can see it. Evaluators can edit the questions, answers, and marks.

3.2 Hardware Interfaces

For the hardware interfaces, as the system is being developed as a web application and mobile application there are no restrictions to access the web application, but the mobile application is only being developed for the Android OS. It is also expected that the user will have a general idea on how to use the mobile device camera as the mobile app will use it to scan documents.

3.3 Software Interfaces

For the web application part, we’ll be using basic HTML, CSS, Django, and Bootstrap. For the mobile version on the app, we’ll be using android. Communications Interfaces. The project will run on all web browsers as well as its app Communications Interfaces.
